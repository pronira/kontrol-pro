ЩО ЗМІНЕНО (тільки functions/index.js — сканер):

1. GOOGLE NEWS 503 (Гранти Україна, Гранти громади, Відновлення,
   Ветерани, Конкурси, Мікрогранти — всі давали HTTP 503):
   → Додано ротацію 4 User-Agent + пауза між спробами.
   → Тепер при 503/403/429 сканер пробує інший UA, а не падає.
   ПІСЛЯ ДЕПЛОЮ: зніми ці джерела з паузи (статус → active).

2. МЕРТВІ ДОМЕНИ (DNS):
   - grant.av.ua (ENOTFOUND) → автопауза після 3 спроб
   - mercycorps.org.ua (ENOTFOUND) → автопауза
   - getgrant.com.ua (ENOTFOUND) → є дубль getgrant.ua, який працює
   → DNS-помилки тепер паузять джерело швидше (3 замість 7).

3. HTTP 403 (devex.com, undp.org, mindev.gov.ua):
   → Ротація UA обходить частину блокувань.
   → Якщо все одно 403 — не валить весь скан, пише зрозумілу помилку.

4. non_ua ФІЛЬТР БУВ НАДТО АГРЕСИВНИЙ (mindev відкидав 18 записів,
   getgrant — 4, бо плутав навігацію з контентом):
   → Тепер відсіює ТІЛЬКИ якщо заголовок ТОЧНО = службове слово
     ("Новини", "Контакти", "Про нас"...), а не якщо просто містить.

5. TELEGRAM 0 РЕЗУЛЬТАТІВ (УКФ, USAID, UNDP, ГУРТ, Простір, ІСАР):
   → Автонормалізація t.me/Channel → t.me/s/Channel (web preview).
   → Деякі канали справді закриті/перейменовані — їх треба замінити
     вручну (Telegram блокує web-preview для приватних каналів).

6. ДУБЛІ БЕЗ НОВИХ (House of Europe, Grant.Market, Granty.org.ua,
   МФВ — знаходили багато, але все дублі):
   → Це нормально якщо нових грантів не з'явилось.
   → Мульти-грант парсинг збережено (3 способи: CSS-блоки,
     посилання в контенті, заголовки H2/H3).

7. ВСІ 8 CLOUD FUNCTIONS на місці:
   scanScheduled, scanSource, scanAll, rejectDetected,
   clearScanLogs, healthCheck, dailyFoundCounter, dailyDetectedCount
